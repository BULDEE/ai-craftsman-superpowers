# v2.6.0 Community Ecosystem Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Enable community pack creation with validation tooling, external pack loading, documentation, and starter skeletons for Go, Rust, and Python.

**Architecture:** Extends the pack-loader from v2.4.0 to scan external directories declared in `.craft-config.yml`. A validate-pack script provides a quality gate for pack authors. The scaffold command gains a `pack` type to generate new pack structures. Community skeletons in `examples/` demonstrate real-world pack patterns.

**Tech Stack:** Bash (pack-loader, validate-pack, scaffold), Markdown (docs, commands, skeletons), YAML (pack.yml)

**Spec reference:** `docs/superpowers/specs/2026-03-29-v2.3-v2.6-core-pack-architecture-design.md` — Section 6

- [ ] **Step 3: Write validate-pack.sh**

Create `scripts/validate-pack.sh`:

```bash
#!/usr/bin/env bash
# =============================================================================
# validate-pack.sh — Validates a pack directory against conventions
#
# Usage: validate-pack.sh <pack-dir> [--check-collisions <packs-root>]
#
# Checks:
#   - pack.yml exists and has required fields (name, version, description, compatibility)
#   - All referenced files exist (validators, agents, SA tools)
#   - No exit 1 in validators (must use exit 0 or exit 2)
#   - Agents have allowedTools defined
#   - Rule IDs don't collide with other packs (optional --check-collisions)
#
# Exit codes:
#   0 = valid
#   2 = validation errors found
# =============================================================================
set -uo pipefail

PACK_DIR=""
CHECK_COLLISIONS=""
COLLISIONS_ROOT=""
ERRORS=0
WARNINGS=0

# Parse args
while [[ $# -gt 0 ]]; do
    case "$1" in
        --check-collisions)
            CHECK_COLLISIONS=true
            COLLISIONS_ROOT="$2"
            shift 2
            ;;
        *)
            PACK_DIR="$1"
            shift
            ;;
    esac
done

if [[ -z "$PACK_DIR" ]]; then
    echo "Usage: validate-pack.sh <pack-dir> [--check-collisions <packs-root>]"
    exit 2
fi

_error() { echo "  ERROR: $1"; ERRORS=$((ERRORS + 1)); }
_warn()  { echo "  WARN:  $1"; WARNINGS=$((WARNINGS + 1)); }
_ok()    { echo "  OK:    $1"; }

# Extract a top-level scalar value from a simple YAML file.
_yml_value() {
    local key="$1" file="$2"
    grep -E "^[[:space:]]*${key}:" "$file" 2>/dev/null | head -1 \
        | sed -E 's/^[^:]+:[[:space:]]*//' | tr -d '"' | tr -d "'" \
        | sed -E 's/^[[:space:]]+//;s/[[:space:]]+$//'
}

# Extract an inline YAML array nested under a parent key.
_yml_nested_array() {
    local parent="$1" child="$2" file="$3"
    local in_parent=false
    while IFS= read -r line; do
        if echo "$line" | grep -qE "^${parent}:"; then
            in_parent=true
            continue
        fi
        if [[ "$in_parent" == true ]]; then
            if echo "$line" | grep -qE '^[a-zA-Z]'; then
                in_parent=false
                continue
            fi
            if echo "$line" | grep -qE "^[[:space:]]+${child}:"; then
                echo "$line" \
                    | sed -E 's/^[^[]*\[//' \
                    | sed -E 's/\].*//' \
                    | tr ',' '\n' \
                    | sed -E 's/^[[:space:]]*"?//;s/"?[[:space:]]*$//' \
                    | grep -v '^$'
                return
            fi
        fi
    done < "$file"
}

# Extract a top-level inline array.
_yml_array() {
    local key="$1" file="$2"
    local line
    line=$(grep -E "^[[:space:]]*${key}:" "$file" 2>/dev/null | head -1)
    [[ -z "$line" ]] && return
    echo "$line" \
        | sed -E 's/^[^[]*\[//' \
        | sed -E 's/\].*//' \
        | tr ',' '\n' \
        | sed -E 's/^[[:space:]]*"?//;s/"?[[:space:]]*$//' \
        | grep -v '^$'
}

MANIFEST="$PACK_DIR/pack.yml"

echo "=== Validating pack: $PACK_DIR ==="

# ---- 1. pack.yml existence ----
if [[ ! -f "$MANIFEST" ]]; then
    _error "pack.yml not found in $PACK_DIR"
    echo ""
    echo "=== Validation: $ERRORS errors, $WARNINGS warnings ==="
    exit 2
fi
_ok "pack.yml exists"

# ---- 2. Required fields ----
for field in name version description; do
    val=$(_yml_value "$field" "$MANIFEST")
    if [[ -n "$val" ]]; then
        _ok "Field '$field' present: $val"
    else
        _error "Required field '$field' missing in pack.yml"
    fi
done

# Check compatibility section
compat_stack=$(_yml_nested_array "compatibility" "stack" "$MANIFEST")
if [[ -n "$compat_stack" ]]; then
    _ok "compatibility.stack defined"
else
    _error "compatibility.stack missing — must declare supported stacks"
fi

# ---- 3. Referenced files exist ----
echo ""
echo "--- File references ---"

# Validators
validators=$(_yml_nested_array "hooks" "validators" "$MANIFEST")
while IFS= read -r v; do
    [[ -z "$v" ]] && continue
    if [[ -f "$PACK_DIR/$v" ]]; then
        _ok "Validator exists: $v"
    else
        _error "Referenced validator not found: $v"
    fi
done <<< "$validators"

# Static analysis tools
sa_tools=$(_yml_nested_array "static_analysis" "tools" "$MANIFEST")
while IFS= read -r t; do
    [[ -z "$t" ]] && continue
    if [[ -f "$PACK_DIR/$t" ]]; then
        _ok "SA tool exists: $t"
    else
        _error "Referenced SA tool not found: $t"
    fi
done <<< "$sa_tools"

# Agents
agents=$(_yml_array "agents" "$MANIFEST")
while IFS= read -r a; do
    [[ -z "$a" ]] && continue
    if [[ -f "$PACK_DIR/$a" ]]; then
        _ok "Agent exists: $a"
    else
        _error "Referenced agent not found: $a"
    fi
done <<< "$agents"

# ---- 4. Validator exit code check ----
echo ""
echo "--- Validator conventions ---"

while IFS= read -r v; do
    [[ -z "$v" ]] && continue
    local_path="$PACK_DIR/$v"
    [[ ! -f "$local_path" ]] && continue
    if grep -qE '^\s*exit\s+1\b' "$local_path" 2>/dev/null; then
        _error "Validator $v contains 'exit 1' — validators must use exit 0 (pass) or exit 2 (block)"
    else
        _ok "Validator $v: no exit 1 found"
    fi
done <<< "$validators"

# ---- 5. Agent allowedTools check ----
echo ""
echo "--- Agent conventions ---"

while IFS= read -r a; do
    [[ -z "$a" ]] && continue
    local_path="$PACK_DIR/$a"
    [[ ! -f "$local_path" ]] && continue
    if grep -qi "allowedTools" "$local_path" 2>/dev/null; then
        _ok "Agent $a has allowedTools"
    else
        _warn "Agent $a missing allowedTools — recommended for security"
    fi
done <<< "$agents"

# ---- 6. Rule ID collision check (optional) ----
if [[ "$CHECK_COLLISIONS" == true && -n "$COLLISIONS_ROOT" ]]; then
    echo ""
    echo "--- Rule ID collision check ---"

    pack_name=$(_yml_value "name" "$MANIFEST")
    pack_rules=$(_yml_nested_array "rules" "builtin" "$MANIFEST")

    while IFS= read -r rule; do
        [[ -z "$rule" ]] && continue
        # Check all other packs
        for other_yml in "$COLLISIONS_ROOT"/*/pack.yml; do
            [[ ! -f "$other_yml" ]] && continue
            other_name=$(_yml_value "name" "$other_yml")
            [[ "$other_name" == "$pack_name" ]] && continue
            other_rules=$(_yml_nested_array "rules" "builtin" "$other_yml")
            if echo "$other_rules" | grep -qx "$rule"; then
                _error "Rule ID collision: '$rule' already defined in pack '$other_name'"
            fi
        done
    done <<< "$pack_rules"

    if [[ $ERRORS -eq 0 ]]; then
        _ok "No rule ID collisions found"
    fi
fi

# ---- Summary ----
echo ""
echo "=== Validation: $ERRORS errors, $WARNINGS warnings ==="

if [[ $ERRORS -gt 0 ]]; then
    exit 2
else
    exit 0
fi
```

# Parse external pack paths from .craft-config.yml
# Returns one path per line (resolved ~ to $HOME)
config_external_packs() {
    local config_file="$PWD/.craft-config.yml"
    [[ ! -f "$config_file" ]] && return

    local in_external=false
    while IFS= read -r line; do
        if echo "$line" | grep -qE '^[[:space:]]+external:'; then
            in_external=true
            continue
        fi
        if [[ "$in_external" == true ]]; then
            # Exit nested block when we hit a non-indented or less-indented line
            if echo "$line" | grep -qE '^[a-zA-Z]' || echo "$line" | grep -qE '^[[:space:]]{0,3}[a-zA-Z]'; then
                in_external=false
                continue
            fi
            # Parse "- path: /some/dir" or "- path: ~/packs/go"
            local path_val
            path_val=$(echo "$line" | grep -oE 'path:[[:space:]]*.*' | sed -E 's/^path:[[:space:]]*//' | tr -d '"' | tr -d "'")
            if [[ -n "$path_val" ]]; then
                # Resolve ~ to $HOME
                path_val="${path_val/#\~/$HOME}"
                echo "$path_val"
            fi
        fi
    done < "$config_file"
}

pack_loader_init() {

    # 1. Load internal packs
    if [[ -d "$packs_dir" ]]; then

#!/usr/bin/env bash
# =============================================================================
# Go Regex Validator — Go Pack
# Provides pack_validate_go() for the pack-loader pipeline.
#
# Rules: GO001-003, WARN-GO001
# Requires: add_violation(), add_warning(), line_has_ignore(), metrics_record_violation()
# =============================================================================

pack_validate_go() {
    local file="$1"

    # GO001: No naked returns in functions with named return values
    # (simplified — real implementation would use AST)
    if grep -qE '^\s+return$' "$file" 2>/dev/null; then
        if grep -qE 'func.*\)\s*\(' "$file" 2>/dev/null; then
            add_warning "GO001" "Naked return found — consider explicit returns for clarity"
        fi
    fi

    # GO002: Error not checked (simplistic regex — catches common pattern)
    local ln_num=0
    while IFS= read -r line; do
        ln_num=$((ln_num + 1))
        # Pattern: assignment without error variable (e.g., "result := doSomething()" without ", err")
        if echo "$line" | grep -qE ':=.*\(.*\)' 2>/dev/null; then
            if ! echo "$line" | grep -qE ', err' 2>/dev/null; then
                if echo "$line" | grep -qE '(Open|Read|Write|Close|Exec|Query|Parse|Decode|Encode|Marshal|Unmarshal)\(' 2>/dev/null; then
                    if ! line_has_ignore "$line" "no-err-check"; then
                        add_violation "GO002" "Possible unchecked error on I/O operation (line $ln_num)"
                    fi
                fi
            fi
        fi
    done < "$file"

    # GO003: No init() functions (prefer explicit initialization)
    if grep -qE '^func init\(\)' "$file" 2>/dev/null; then
        add_violation "GO003" "init() function found — prefer explicit initialization"
    fi

    # WARN-GO001: Function too long (>50 lines)
    local func_start=0
    local in_func=false
    ln_num=0
    while IFS= read -r line; do
        ln_num=$((ln_num + 1))
        if echo "$line" | grep -qE '^func ' 2>/dev/null; then
            in_func=true
            func_start=$ln_num
        fi
        if [[ "$in_func" == true ]] && echo "$line" | grep -qE '^\}$' 2>/dev/null; then
            local func_len=$((ln_num - func_start))
            if [[ $func_len -gt 50 ]]; then
                add_warning "WARN-GO001" "Function starting at line $func_start is $func_len lines — consider splitting"
            fi
            in_func=false
        fi
    done < "$file"
}

#!/usr/bin/env bash
# =============================================================================
# Rust Regex Validator — Rust Pack
# Provides pack_validate_rust() for the pack-loader pipeline.
#
# Rules: RUST001-003, WARN-RUST001
# Requires: add_violation(), add_warning(), line_has_ignore()
# =============================================================================

pack_validate_rust() {
    local file="$1"

    # RUST001: No unwrap() in production code (except tests)
    if [[ "$file" != *"test"* && "$file" != *"_test.rs"* ]]; then
        local ln_num=0
        while IFS= read -r line; do
            ln_num=$((ln_num + 1))
            if echo "$line" | grep -qE '\.unwrap\(\)' 2>/dev/null; then
                if ! line_has_ignore "$line" "allow-unwrap"; then
                    add_violation "RUST001" ".unwrap() found — use ? operator or proper error handling (line $ln_num)"
                fi
            fi
        done < "$file"
    fi

    # RUST002: No panic! in library code
    if [[ "$file" != *"main.rs"* && "$file" != *"test"* ]]; then
        if grep -qE 'panic!\(' "$file" 2>/dev/null; then
            add_violation "RUST002" "panic!() in library code — return Result instead"
        fi
    fi

    # RUST003: No clone() without justification
    local clone_count
    clone_count=$(grep -cE '\.clone\(\)' "$file" 2>/dev/null || echo "0")
    if [[ "$clone_count" -gt 3 ]]; then
        add_warning "RUST003" "Found $clone_count .clone() calls — review ownership patterns"
    fi

    # WARN-RUST001: Function too long (>40 lines for Rust)
    local func_start=0
    local brace_depth=0
    local in_func=false
    local ln_num=0
    while IFS= read -r line; do
        ln_num=$((ln_num + 1))
        if echo "$line" | grep -qE '^\s*(pub\s+)?(async\s+)?fn\s+' 2>/dev/null; then
            in_func=true
            func_start=$ln_num
            brace_depth=0
        fi
        if [[ "$in_func" == true ]]; then
            local opens closes
            opens=$(echo "$line" | tr -cd '{' | wc -c)
            closes=$(echo "$line" | tr -cd '}' | wc -c)
            brace_depth=$((brace_depth + opens - closes))
            if [[ $brace_depth -le 0 && $func_start -gt 0 ]]; then
                local func_len=$((ln_num - func_start))
                if [[ $func_len -gt 40 ]]; then
                    add_warning "WARN-RUST001" "Function at line $func_start is $func_len lines — consider extracting"
                fi
                in_func=false
            fi
        fi
    done < "$file"
}

use std::fmt;

/// Domain error for order operations.
#[derive(Debug)]
pub enum OrderError {
    NotFound(String),
    InvalidState(String),
    PersistenceError(String),
}

impl fmt::Display for OrderError {
    fn fmt(&self, f: &mut fmt::Formatter<'_>) -> fmt::Result {
        match self {
            Self::NotFound(id) => write!(f, "order not found: {id}"),
            Self::InvalidState(msg) => write!(f, "invalid order state: {msg}"),
            Self::PersistenceError(msg) => write!(f, "persistence error: {msg}"),
        }
    }
}

impl std::error::Error for OrderError {}

/// Repository trait — domain boundary, no implementation details.
pub trait OrderRepository: Send + Sync {
    fn find_by_id(&self, id: &str) -> Result<Order, OrderError>;
    fn save(&self, order: &Order) -> Result<(), OrderError>;
}

/// Order aggregate — private fields, behavioral methods.
pub struct Order {
    id: String,
    customer_id: String,
    status: OrderStatus,
    items: Vec<OrderItem>,
}

impl Order {
    /// Factory method — the only way to create an Order.
    pub fn create(customer_id: String, items: Vec<OrderItem>) -> Result<Self, OrderError> {
        if items.is_empty() {
            return Err(OrderError::InvalidState("order must have at least one item".into()));
        }
        Ok(Self {
            id: uuid::Uuid::new_v4().to_string(),
            customer_id,
            status: OrderStatus::Pending,
            items,
        })
    }

    pub fn confirm(&mut self) -> Result<(), OrderError> {
        if self.status != OrderStatus::Pending {
            return Err(OrderError::InvalidState(
                format!("cannot confirm order in {:?} status", self.status),
            ));
        }
        self.status = OrderStatus::Confirmed;
        Ok(())
    }

    pub fn id(&self) -> &str { &self.id }
    pub fn status(&self) -> &OrderStatus { &self.status }
}

    ln_num=0

Demonstrates: type hints, dataclass, explicit error handling, no mutable defaults.

from __future__ import annotations

from dataclasses import dataclass
from typing import Protocol


class OrderRepository(Protocol):
    """Repository interface — domain boundary."""

    def find_by_id(self, order_id: str) -> Order | None: ...
    def save(self, order: Order) -> None: ...


@dataclass(frozen=True)
class CreateOrderCommand:
    """Immutable command — no setters, no mutation."""

    items: list[OrderItem]

    """Use case handler — single responsibility."""

        if not command.items:
            raise ValueError("Order must have at least one item")

        order = Order.create(
            customer_id=command.customer_id,
            items=command.items,
        )
        self._repo.save(order)

## Writing a Validator

Validators are bash scripts that define a `pack_validate_<lang>()` function. The pack-loader sources your script and calls this function for each file of the matching language.

### Available Helper Functions

Your validator receives these functions from the orchestrator:

| Function | Purpose |
|----------|---------|
| `add_violation "RULE_ID" "message"` | Report a blocking violation |
| `add_warning "RULE_ID" "message"` | Report a non-blocking warning |
| `line_has_ignore "$line" "ignore-tag"` | Check if a line has `craftsman-ignore: <tag>` |
| `metrics_record_violation ...` | Record in metrics DB (optional) |

### Naming Conventions for Rule IDs

Use a unique prefix to avoid collisions with other packs:

| Pack | Prefix | Example |
|------|--------|---------|
| symfony | PHP, LAYER, PHPSTAN, DEPTRAC | PHP001, LAYER001 |
| react | TS, ESLINT | TS001, ESLINT001 |
| go | GO | GO001 |
| rust | RUST | RUST001 |
| python | PY | PY001 |

Warnings use the `WARN-` prefix: `WARN-GO001`, `WARN-PY001`.

### Validator Template

```bash
#!/usr/bin/env bash
pack_validate_<lang>() {
    local file="$1"

    # MYPACK001: Description
    if grep -q "bad_pattern" "$file" 2>/dev/null; then
        add_violation "MYPACK001" "Descriptive error message"
    fi

    # WARN-MYPACK001: Soft rule
    if grep -q "questionable_pattern" "$file" 2>/dev/null; then
        add_warning "WARN-MYPACK001" "Consider improving this pattern"
    fi
}
```

### Rules

- **Never use `exit 1`** — validators must use `exit 0` (pass) or `exit 2` (block)
- Always redirect stderr: `2>/dev/null` on grep/sed calls
- Use `line_has_ignore` to respect `craftsman-ignore` comments
- Keep validators fast (<50ms per file) — regex only, no external tool calls
- Static analysis tools (eslint, phpstan, clippy) go in `static-analysis/`, not validators
