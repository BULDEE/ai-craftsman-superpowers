## Step 2: Quick Setup Check

Check if already configured:

!`[[ -f .craft-config.yml ]] && echo "CONFIG_EXISTS=true" || echo "CONFIG_EXISTS=false"`

### If not configured

Offer quick setup:

**Auto-detect stack:**

!`{ [[ -f composer.json ]] && echo "PHP=true" || echo "PHP=false"; } && { [[ -f package.json ]] && echo "NODE=true" || echo "NODE=false"; } 2>/dev/null || echo "PHP=false NODE=false"`

Based on detection:
- PHP only → suggest Symfony Pack
- Node only → suggest React Pack
- Both → suggest both packs
- Neither → suggest Core pack only

Ask the user:

```
Quick setup (30 seconds):
  Detected stack: <PHP/Node/both/unknown>

  Strictness preference?
    1. strict    — All rules enforced, hooks block violations
    2. moderate  — Warnings for most rules, blocks only critical
    3. relaxed   — Advisory mode only, no blocking
```

Generate a minimal `.craft-config.yml` using the Write tool:

```yaml
# AI Craftsman Superpowers — Project Configuration
# Generated by /craftsman:start
# Full setup: /craftsman:setup

version: "1.0"

packs:
  core: true
  symfony: <true/false based on detection>
  react: <true/false based on detection>
  ai: false

rules:
  strictness: <strict|moderate|relaxed>
  php:
    final_classes: true
    strict_types: true
    no_setters: true
  typescript:
    no_any: true
    readonly_default: true
```

Display:
```
Config created at .craft-config.yml
Run /craftsman:setup for full configuration (DISC profile, bias detection, etc.)
```

### If already configured

Read and display:

!`cat .craft-config.yml 2>/dev/null | grep -E "^(version|  symfony|  react|  ai|  strictness)" || echo "Config found."`

Display:
```
Configuration already exists. Continuing to codebase analysis.
Run /craftsman:setup to reconfigure.
```

## Step 3: Codebase Scan

Run a quick analysis of the project:

**File counts by type:**

!`echo "PHP: $(find src -name '*.php' 2>/dev/null | wc -l | tr -d ' ') files" && echo "TypeScript: $(find src -name '*.ts' -o -name '*.tsx' 2>/dev/null | wc -l | tr -d ' ') files" && echo "Tests: $(find . -name '*Test.php' -o -name '*.test.ts' -o -name '*.spec.ts' 2>/dev/null | grep -v vendor | grep -v node_modules | wc -l | tr -d ' ') test files" 2>/dev/null || echo "No source files detected yet."`

**DDD layer detection:**

!`for layer in Domain Application Infrastructure Presentation; do count=$(find src/$layer -type f 2>/dev/null | wc -l | tr -d ' '); [[ "$count" -gt "0" ]] && echo "src/$layer: $count files"; done 2>/dev/null || echo "No DDD layers detected."`

**Quality tooling check:**

!`{ command -v phpstan >/dev/null 2>&1 && echo "PHPStan: installed" || { [[ -f vendor/bin/phpstan ]] && echo "PHPStan: installed (vendor)" || echo "PHPStan: not found"; }; } ; { [[ -f .eslintrc* ]] || [[ -f eslint.config.*  ]] && echo "ESLint: configured" || echo "ESLint: not configured"; } ; { [[ -f phpunit.xml ]] || [[ -f phpunit.xml.dist ]] && echo "PHPUnit: configured" || echo "PHPUnit: not configured"; } ; { [[ -f jest.config.* ]] && echo "Jest: configured" || echo "Jest: not configured"; } 2>/dev/null || echo "Tooling check failed."`

Display findings in a readable format:

```
Codebase Analysis:
<PHP/TS file counts>
<DDD layer breakdown if present>
<Quality tooling found>
```

## Step 4: Skill Recommendations

Based on the codebase scan, suggest the 5 most relevant skills.

**Logic:**
- DDD layers detected → suggest `/craftsman:entity`, `/craftsman:design`
- PHP files detected → suggest `/craftsman:verify`, `/craftsman:challenge`
- TypeScript files detected → suggest `/craftsman:component`
- Test files > 0 → suggest `/craftsman:test`
- Any code present → always suggest `/craftsman:debug`, `/craftsman:plan`

Display:

```
Recommended Skills for Your Project:
1. /craftsman:<skill>   — <reason based on detection>
2. /craftsman:<skill>   — <reason>
3. /craftsman:<skill>   — <reason>
4. /craftsman:<skill>   — <reason>
5. /craftsman:<skill>   — <reason>
```
